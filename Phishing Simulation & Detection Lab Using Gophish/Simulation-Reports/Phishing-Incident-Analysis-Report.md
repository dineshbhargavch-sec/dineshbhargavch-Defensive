# Phishing Incident Analysis Report

## Incident Summary
A simulated phishing campaign was conducted using Gophish to replicate common credential-harvesting attacks.

## Attack Vector
- **Email Type:** Security Alert / Account Verification
- **Technique:** Social engineering using urgency and authority
- **Landing Page:** Fake login portal hosted in a controlled environment

## User Interaction Timeline
1. Email delivered to user inbox.
2. User opened the email.
3. User clicked malicious link.
4. User submitted credentials on phishing landing page.

## Indicators of Phishing
- Suspicious sender domain
- Generic greeting ("Dear User")
- Urgent call-to-action
- Embedded external link

## Impact Analysis
- No real credentials compromised (simulation).
- Demonstrated potential exposure if attack was real.

## Mitigation Actions
- Campaign stopped immediately after data collection.
- Results documented for awareness training.

## Lessons Learned
- Users often fail to inspect sender addresses.
- Time-pressure messaging significantly increases success rates.
